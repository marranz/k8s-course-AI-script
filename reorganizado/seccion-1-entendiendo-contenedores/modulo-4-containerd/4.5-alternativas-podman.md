# 4.5 Alternativas: Podman y Otras Opciones

## Podman

### ¿Qué es Podman?

Podman es una alternativa a Docker que:
- **Daemonless**: No requiere daemon corriendo
- **Rootless**: Ejecuta sin privilegios root por defecto
- **Compatible**: CLI compatible con Docker
- **Pods**: Soporte nativo de pods (como Kubernetes)

### Instalación

\`\`\`bash
# macOS
brew install podman
podman machine init
podman machine start

# Linux
sudo apt-get install podman

# Verificar
podman --version
\`\`\`

### Uso Básico

\`\`\`bash
# Mismo que Docker
podman run -d -p 8080:80 nginx
podman ps
podman images
podman logs <container>

# Alias para compatibilidad
alias docker=podman
\`\`\`

### Ventajas de Podman

1. **Seguridad**: Rootless por defecto
2. **Sin daemon**: Menos overhead
3. **Pods**: Grupos de contenedores
4. **Kubernetes**: Genera YAML de K8s

### Ejemplo de Pod

\`\`\`bash
# Crear pod
podman pod create --name mypod -p 8080:80

# Añadir contenedores al pod
podman run -d --pod mypod nginx
podman run -d --pod mypod redis

# Listar pods
podman pod ps
\`\`\`

### Generar Kubernetes YAML

\`\`\`bash
podman generate kube mypod > pod.yaml
\`\`\`

## nerdctl

Cliente containerd compatible con Docker:

\`\`\`bash
# Instalar
brew install nerdctl  # macOS

# Uso (igual que Docker)
nerdctl run -d -p 8080:80 nginx
nerdctl compose up
\`\`\`

## Comparación

| Feature | Docker | Podman | nerdctl |
|---------|--------|--------|---------|
| Daemon | Sí | No | No |
| Root | Sí | Rootless | Depende |
| Pods | No | Sí | No |
| Compose | Sí | Sí | Sí |
| Desktop | Sí | Sí | No |

## Cuándo Usar Cada Uno

### Docker
- Desarrollo local
- Ecosistema maduro
- Docker Desktop

### Podman
- Seguridad (rootless)
- Sin daemon
- Entornos empresariales grandes

### nerdctl
- Usar containerd directamente
- Compatible con Docker
- Lightweight
